================================================================================
DFU (FIRMWARE UPDATE) FLOW THROUGH MODULES
================================================================================

1. OTA FLAG CHECK
   Module: dfu_crc
   Function: is_need_updata_image (FUN_0042e170)
   - Checks magic value 0x55555555 in persistent storage
   - Returns: 0 = no update, 1 = update pending
   - Set by main firmware before reset when OTA file received

2. OTA DECISION (thread_manager_task)
   Module: thread_manager
   Function: thread_manager_task (FUN_0042e244)
   - If flag == 0: sends msg type 0 -> DFU task jumps to app immediately
     Log: "DO NOT need updata firmware, GO TO APP"
   - If flag != 0: sends msg type 1 -> DFU task processes OTA image
     Log: "need updata firmware"

3. DFU MESSAGE DISPATCH
   Module: dfu_core
   Function: dfu_task_msg_send (FUN_0042dbee)
   - Pushes 40-byte message to DFU queue (osMessageQueuePut)
   - Sets thread flag 0x400000 on DFU thread (osThreadFlagsSet)

4. DFU TASK PROCESSING
   Module: dfu_core
   Function: thread_dfu_task (FUN_0042dc60) -> _thread_msg_handler (FUN_0042dda4)

   4a. OPEN OTA FILE
       Module: dfu_file
       Function: dfu_file_open -> lfs_fopen("ota/s200_firmware_ota.bin", "r")
       - Opens EVENOTA container file from LittleFS

   4b. READ & PARSE HEADER
       Module: dfu_file
       Functions: dfu_file_read_header, dfu_file_parse_metadata
       - Reads 32-byte OTA header: magic, version, blob count, entry table
       - dfu_file_validate_magic: checks EVENOTA magic number
       - dfu_file_check_version: validates firmware version
       - dfu_file_get_blob_size: reads total image size
       - dfu_file_log_metadata: prints header details via elog
         Log: "magicNum", "filesize", "blobSize"

   4c. PER-COMPONENT INSTALLATION LOOP
       Module: dfu_file
       Function: dfu_file_install_firmware (FUN_0042c644)
       Log: "Installing ...", "Installing %d(%d)"

       For each of the 6 EVENOTA entries:
         i.   dfu_file_get_entry_info: read entry offset, size, target addr
         ii.  dfu_file_extract_component: seek to entry data
         iii. dfu_file_verify_entry_crc: CRC-32 check
              Module: dfu_crc -> crc32_update (table-driven CRC-32)
              Module: dfu_core -> app_dfu_image_crc_check
         iv.  dfu_install_flash_erase_region:
              Module: dfu_core -> dfu_flash_erase_range
              Module: drv_norflash -> mx25u25643g_sector_erase (4KB sectors)
         v.   dfu_install_flash_write:
              Module: dfu_core -> app_dfu_system_program
              Module: drv_norflash -> mx25u25643g_page_program (256B pages)
         vi.  dfu_install_verify:
              Module: dfu_core -> _verifyFlashContent (read-back compare)
              Log: "flash write verify ok/failed"
         vii. dfu_install_progress_report: log progress %

       Log: "---Install end---"

   4d. POST-INSTALL
       Module: dfu_file
       Functions: dfu_install_finalize, dfu_install_mark_complete
       - Clear OTA flag
       - Close OTA file

5. JUMP TO APPLICATION
   Module: dfu_core
   Function: dfu_pre_jump_deinit (FUN_0042dd3e)
   - Disable all peripheral interrupts
   - Flush data caches (am_hal_cachectrl_invalidate)
   - Disable MSPI, UART, I2C peripherals
   - Deinit NOR flash driver

   Module: boot
   Function: boot_jump_to_app (FUN_0042dbdc)
   - Write SCB->VTOR = 0x00438000 (main firmware base)
   - Load SP from vector table[0]
   - Branch to reset vector at vector table[1]
   - Bootloader execution ends; main firmware starts

6. ERROR RECOVERY
   Module: dfu_core
   Function: bootloader_error_reset (FUN_0042dd5a)
   - Log: "Bootloader ota fail, system reset!!!"
   - am_hal_reset_control -> system reset
   - Bootloader restarts from Phase 1

   Module: dfu_core
   Function: dfu_error_halt (FUN_0042e126)
   - Signals cleanup via event flags
   - Enters infinite osDelay (watchdog will eventually reset)
