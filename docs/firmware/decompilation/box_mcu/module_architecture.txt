================================================================================
INTER-MODULE COMMUNICATION PATHS
================================================================================

The box firmware has a relatively flat call graph compared to the main G2
firmware. Here are the primary inter-module communication paths:

1. BOOT CHAIN
   startup -> hal_core -> hal_rcc -> hal_gpio -> app_init -> rtos_init

   system_init_main calls:
     hal_systick_init, hal_rcc_periph_and_clock_init, gpio_and_rcc_init,
     i2c_gpio_reinit, uart_gls_init_with_iwdg, uart_debug_rx_init,
     uart_peripheral_init, self_check_all_peripherals,
     osKernelRun, create_all_tasks_and_timers, osKernelStart

2. GLASSES UART PROTOCOL PATH
   ISR: hal_uart -> uart_protocol (uart_rx_isr_callback)
   Task: uart_protocol (uart_rx_dispatch) -> uart_protocol (glasses_cmd_handler)
   TX: app_* -> uart_tx -> hal_uart (hal_uart_transmit_blocking)

   Frame format: 5A A5 FF [cmd] [data...] [CRC]

   Command dispatch (glasses_cmd_handler @ 0x08000de0):
     0x3D/0x3E -> app_aging (exit aging)
     0x3F      -> gpio_ctrl (enable/disable output)
     0x41      -> cortex_m (system reset)
     0x50      -> app_sn (version query)
     0x51      -> app_sn (status query)
     0x56      -> app_ota (OTA GLS status)
     0x5C      -> app_state (case status query)
     0x5D      -> app_charging (charging control)
     0x5E      -> app_sn (set SN)
     0x5F      -> app_sn (SN query)
     0x68      -> gpio_ctrl (silent mode toggle)

3. SERIAL DEBUG PATH
   ISR: hal_uart -> uart_protocol (debug port)
   Task: uart_protocol (serial_cmd_dispatcher) -> app_debug / pmic / hal_iwdg

   Debug command dispatch (serial_cmd_dispatcher @ 0x08000928):
     0xAF -> app_debug (debug output toggle)
     0xB0 -> i2c_yhm2510 (dump registers via timer)
     0xD0 -> pmic (watchdog toggle)
     0xD1 -> app_debug (simulation modes)
     0xD2/D3 -> app_charging (one-side charging)
     0xDD -> hal_iwdg (dog-feed toggle)
     0xDF -> hal_pwr (ship mode)
     0xEE -> hal_pwr (standby)
     0xFF -> cortex_m (system reset)

4. I2C PERIPHERAL BUS
   app_init -> i2c_periph -> hal_i2c (hardware I2C)
   app_init -> i2c_periph -> i2c_bitbang (software I2C fallback)
   app_init -> i2c_yhm2510 -> i2c_periph -> hal_i2c

   I2C devices:
     YHM2510 (0xC8/0xC9): Battery fuel gauge
       -> yhm2510_read_battery_pct, voltage, temperature, capacity
     PMIC (0x70/0x71): Power management IC
       -> get_pmic_status_byte, pmic_boost_check/enable
     4005: Watchdog controller (chip ID verified at boot)
     2217: Unknown IC (chip ID verified at boot)

5. GLASSES DETECTION PATH
   glsDetectTask -> gpio_ctrl -> adc_periph -> gpio_ctrl
   glsDetectTask -> rtos_wrapper (osDelay, osTimerStart)
   glsDetectTask -> app_debug (debug_printf)

   IR scan cycle:
     gpio_ctrl (drive LED) -> osDelay(3ms) -> adc_periph (read) ->
     threshold check -> update state struct

6. CHARGING CONTROL PATH
   defaultTask -> app_charging (bat_change_handler, decide_charge_side)
   defaultTask -> gpio_ctrl (update_gls_charge_pins, set_5v)
   defaultTask -> i2c_periph (yhm2510_read_battery_pct)
   defaultTask -> pmic (pmic_boost_enable/check)

   Charging thresholds:
     bat >= 0x5A (90%) -> disable charging
     bat >= 98%        -> charging done
     Over-temperature  -> stop PMIC charging

7. OTA UPDATE PATH
   defaultTask -> uart_tx (send OTA check frame)
   defaultTask -> uart_protocol (uart_rx_wait_and_parse for OTA response)
   defaultTask -> app_ota (ota_box_download_firmware)
   app_ota -> hal_flash (erase/program target bank)
   app_ota -> app_sn (copy SN between banks)
   app_ota -> app_ota (ota_swap_bank_and_reset)

   OTA flow:
     1. Check glasses ready (ota_gls_ready L/R/box)
     2. Get running bank (flash_get_running_bank)
     3. Erase target bank (ota_erase_target_bank)
     4. Copy SN (ota_copy_sn_between_banks)
     5. Download firmware (ota_box_download_firmware)
     6. Verify CRC (ota_verify_downloaded_crc)
     7. Notify glasses (ota_notify_gls_result)
     8. Swap bank and reset (ota_swap_bank_and_reset)

8. POWER MANAGEMENT PATH
   defaultTask -> hal_pwr (hal_pwr_enter_standby, hal_pwr_enter_sleep)
   defaultTask -> hal_iwdg (hal_iwdg_refresh before sleep)
   defaultTask -> pmic (watchdog_enable, pmic_boost_enable)

   Standby reasons:
     - Idle (empty box, 30 iterations ~30s)
     - CMD (debug command 0xEE)
     - Low battery (YHM2510 capacity < threshold)
     - Glasses battery full (both sides done)

   Wake sources:
     - HALL sensor (lid open/close)
     - USB (plug/unplug)
     - RTC (periodic wake for monitoring)

9. AGING TEST PATH
   defaultTask -> app_aging (start_aging, aging_running_state_machine)
   app_aging -> i2c_periph (yhm2510 battery queries)
   app_aging -> gpio_ctrl (toggle 5V for aging)
   app_aging -> uart_tx (query glasses aging status)

   Aging states: AGING_PRE -> AGING_RUNNING -> CHARGING_AGING <-> AGING_ONLY -> AGING_DONE

10. EVENT FLAG SIGNALING
    ISR (EXTI) -> rtos_wrapper (osEventFlagsSet) -> defaultTask (osEventFlagsWait)

    Events:
      Hall sensor change  -> "hall int, isOpen:%d"   -> event flag
      USB plug/unplug     -> "USB int, hasUsb:%d"    -> event flag + GLS reset
      Battery change      -> "Bat change: %d -> %d"  -> send 0x13 to glasses
      2217 interrupt      -> "2217 int"              -> (handled inline)

role as a charging controller with no BLE, display, or user-facing features.

Box firmware features mapped to Swift SDK types:

  Box JSON status -> G2GlassesCaseReader (soc, charging, lidOpen, inCase)
    {"vol":%d,"pct":%d,"open":%d,"usb":%d,"cur":%d,"GLS_L":%d,"GLS_R":%d,"temp":%d}

  Box UART protocol -> G2BluetoothClient (0x81-20 BoxDetect service)
    Main FW relays box status via BLE service 0x81-20

  Box OTA -> G2FirmwareAPIClient, G2FirmwareTransferClient
    Box component in EVENOTA container (index 5: "box-case-mcu")

  Box commands -> G2ResponseDecoder (onDisplayTriggerResponse on 0x81-00)
    DisplayTrigger is how main FW signals box presence/absence
