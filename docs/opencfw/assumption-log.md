## Promotion Rule

An assumption may be promoted into code only when one of these is true:

- it is backed by identified functionality,
- it is isolated behind instrumentation or feature flags,
- or it is explicitly required to keep the clean-room architecture coherent.

## Retirement Rule

Retire or rewrite an assumption when:

- a newer RE artifact contradicts it,
- a live capture closes the same behavior more strongly,
- or the implementation needs exact field-level compatibility.
